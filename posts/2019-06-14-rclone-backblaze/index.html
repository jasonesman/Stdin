Backing up NextCloud data with Rclone and Backblaze

We are going to use backblaze b2, cost is better then Amazon S3. To backup the data we are going to use rclone. We are NOT trying to do a full server backup, I can rebuild the server. We are only going to backup the nextcloud data directory.


First we need a couple of items installed.


+ Install some prerequisites: yum install goland git
-You should have setup backblaze b2, and created an application key. If not, I will write this up later.
+ Make a place for Go: mkdir $HOME/gopath
+ Add Go’s folder to your path: export GOPATH="$HOME/gopath/"
+ Install rclone in the GO folder: go get github.com/ncw/rclone
+ Copy the program into your binary folder: cp $GOPATH/bin/rclone /usr/bin/

Now we need to configure rclone to use Backblaze: rclone config


+ Choose New Remote
+ Type Backblaze for remote name
+ Choose 5 for Backblaze B2
+ Enter your Application ID
+ Enter your Application Key
+ Hit N for no
+ Choose Y for yes its ok
+ Choose Quit config


Crontab and schedule


Now that you have it installed we need to setup our script. I made this in /root/backup.sh 

#!/bin/bash

src=/var/www/html/nextcloud/data/
dest=Backblaze:my-bucket-name
log_file=/var/log/rclone.log
log_level=INFO
min_age=15m
transfers=16

/usr/local/bin/rclone sync \
	--copy-links \
	--b2-hard-delete \
	--fast-list \
	--min-age ${min_age} \
	--transfers ${transfers} \
	--log-level ${log_level} \
	--log-file ${log_file} \
	 ${src} ${dest} $@
rc=$?

if [[ $rc != 0 ]]; then
	echo "An error occurred. Please check the logs." 
else
	echo "Backup succeeded" 
fi


Now to make a cronjob for the script to run.
crontab -e
0 5 * * * /root/backup.sh >/dev/null 2>&1
